HTMX

Is it possible to have multiple trigger types in HTMX? How would I use/code the htmx attributes for this scenario? Make sure you respond showing what would be the code for the select element but also remember to mention what does the value "/api/search" represents!

Yes, you can have multiple trigger types. The value of hx-trigger just needs to have the multiple events comma separated, like this:


                <select name="city" id="city"
                        hx-post="/api/search"
                        hx-trigger="load, change"
                        hx-target="#result">
                    <option>San Francisco</option>
                    <option>New York</option>
                    <option>Las Vegas</option>
                </select>

The "/api/search" is the URL that the select value will POST to.
